# Generate Website from BRIEF.md (Old)

Reads BRIEF.md with client data and generates a complete website repo with:

- Unique GitHub repo name
- Populated config/business.ts
- Ready-to-push Git commit
- TypeScript validation

**How to use:**

Say any of:

- `generate website`
- `/generate-website pension`
- `apply the brief`
- `create website for pension`
- `setup website from brief`

**What you provide:**

- BRIEF.md must be filled in with client data

**What you get:**

- New directory ~/Documents/Github/{repo-name}
- config/business.ts with all client data
- Git repo initialized and committed
- Summary with next steps (GitHub push, Vercel import)

---

## Execution Steps

Follow the workflow in `.instructions.md`:

1. **Read BRIEF.md** — extract all business data
2. **Validate** — check types, required fields, formats
3. **Generate repo name** — pattern: `website-[type]-[name-slug]`
4. **Create directory** — copy template files
5. **Populate config/business.ts** — write complete config from BRIEF data
6. **Initialize Git** — commit with repo name as message
7. **Type check** — run tsc to validate no errors
8. **Output summary** — give user next steps

---

## Important Rules

- **Don't change component code** — all site logic is done via config/business.ts
- **Don't modify templates** — Navbar.tsx, Hero.tsx, etc. stay as-is
- **Images:** Keep external URLs if postimg/Unsplash, note that they can move to /public/images/ later
- **No code generation for components** — config only
- **One repo per client** — never monorepo structure
- **Commit message format:** `init: {repo-name}`

## Data Mapping: BRIEF.md → config/business.ts

| BRIEF.md Section | config Field                                                                   | Notes                                                           |
| ---------------- | ------------------------------------------------------------------------------ | --------------------------------------------------------------- |
| Business Type    | `businessType`                                                                 | Must match: pension, hotel, cottage, restaurant, salon, generic |
| Business Info    | `name`, `tagline`, `description`, `phone`, `email`, `address`, `city`, `hours` | Exact copy, no edits                                            |
| Primary color    | `primaryColor`                                                                 | Hex format #XXXXXX                                              |
| Booking URLs     | `bookingComUrl`, `airbnbUrl`                                                   | Only if type is hotel/pension/cottage                           |
| Social Links     | `socialLinks: { facebook?, instagram?, tiktok? }`                              | Optional, all URLs                                              |
| WhatsApp         | `whatsapp`                                                                     | Romanian format: "40722000000" no spaces/+                      |
| Google Maps      | `googleMapsEmbedUrl`                                                           | Extract src URL from iframe tag                                 |
| Hero Photo       | `heroImageUrl`, `heroImageAlt`                                                 | URL + description                                               |
| Services         | `services[]`                                                                   | Array with title, description, optional price                   |
| About Bullets    | `bulletPoints[]`                                                               | Array of 3-4 strings                                            |
| Stats            | `stats[]`                                                                      | Array with value + label                                        |
| Rooms            | `rooms[]`                                                                      | Only if type is hotel/pension/cottage                           |
| Testimonials     | `testimonials[]`                                                               | Array with name, role, stars (1-5), text                        |
| FAQ              | `faq[]`                                                                        | Array with question + answer                                    |
| Gallery          | `gallery[]`                                                                    | Array of image URLs                                             |

## Design Variant Assignment

Default to `"classic"` unless overridden.

Can be customized per client type:

- Pension: `classic` (warm, traditional)
- Hotel: `dark` (upscale)
- Restaurant: `dark` (elegant)
- Salon: `bold` (modern)
- Generic: `classic`

## Error Handling

**If BRIEF.md is incomplete:**

- List missing required fields (name, type, phone, email, city, color)
- Don't proceed — ask user to fill in BRIEF.md first

**If business type is invalid:**

- Show valid options: pension, hotel, cottage, restaurant, salon, generic
- Ask user to fix BRIEF.md

**If images are broken URLs:**

- Note it in the output but don't block
- Suggest user can add local images to /public/images/ later

**If TypeScript fails:**

- Show error
- Check config/business.ts syntax (quotes, brackets, commas)
- Fix and retry
