# Website Generation Instructions

**Trigger:** User says "generate website" or "/generate-website [type]"

**Purpose:** Read BRIEF.md, create a new GitHub repo with a unique name, populate config/business.ts with client data, and prepare the site for deployment.

---

## Pre-Generation Checklist

- [ ] BRIEF.md is filled in completely (or mostly — optional fields can be empty)
- [ ] All images are either:
  - In `/public/images/[client-name]/` (preferred)
  - External URLs (postimg, Unsplash, etc. — fine for now, but will be moved to repo later)
- [ ] Video (if any) is on YouTube or will be added later

---

## Generation Workflow

### Step 1: Read and Validate BRIEF.md

Parse BRIEF.md and extract:
- Business name, type, location, contact info
- Services, rooms, testimonials, FAQ, gallery
- Booking URLs (Booking.com, Airbnb)
- Social links, WhatsApp, Maps embed
- Hero image, colors, all text content

**Validate:**
- Business type is valid: `pension | hotel | cottage | restaurant | salon | generic`
- Phone number is not empty
- Primary color is valid hex (e.g., `#1a5276`)

---

### Step 2: Generate Unique GitHub Repo Name

Pattern: `website-[business-type]-[business-name-slug]`

Examples:
- Pension GreenHouse → `website-pension-greenhouse`
- Azur Restaurant → `website-restaurant-azur`
- The Fade Room → `website-salon-the-fade-room`

**Rules:**
- Lowercase
- Replace spaces with hyphens
- Remove special characters (accents, apostrophes)
- Max 50 chars

---

### Step 3: Create New Directory Structure

```bash
mkdir -p ~/Documents/Github/{repo-name}
cd ~/Documents/Github/{repo-name}
```

Copy entire template:
```bash
cp -r ~/Documents/Github/website-template/* ~/Documents/Github/{repo-name}/
```

Create images folder:
```bash
mkdir -p public/images/{client-name-slug}
```

---

### Step 4: Populate config/business.ts

Read BRIEF.md and generate a complete `export const business: BusinessConfig = { ... }` object with:

- `name`, `tagline`, `description`
- `phone`, `email`, `address`, `city`, `hours`
- `primaryColor`, `businessType`, `designVariant`
- `heroImageUrl`, `heroImageAlt`
- `bookingComUrl` (if type is hotel/pension/cottage)
- `socialLinks` (facebook, instagram, tiktok)
- `whatsapp` (if provided, in Romanian format: "40722000000")
- `googleMapsEmbedUrl` (if provided, extract src URL from iframe)
- `services[]` (with title, description, optional price)
- `rooms[]` (if type is hotel/pension/cottage — with title, description, price, features, imageUrl)
- `testimonials[]` (if present — with name, role, stars, text)
- `faq[]` (if present — with question, answer)
- `gallery[]` (if present — array of image URLs)
- `bulletPoints[]`, `stats[]` (if provided in About section)

**Rules:**
- All text fields should be exactly as in BRIEF.md (no paraphrasing)
- Remove any placeholder text like "Se va completa..."
- If a section is empty/blank in BRIEF.md, omit it from the config (optional sections)
- Ensure all strings are properly escaped (quotes, newlines)
- Type check: `designVariant` should be `"classic"` unless specified otherwise

---

### Step 5: Handle Images

**If images are in BRIEF.md as URLs:**

- If they're external URLs (postimg, Unsplash) — keep them as-is in config
- If they're postimg.cc links — **note for user:** These should be replaced with local images later

**If images will be added locally:**

1. User will provide images (JPG/PNG)
2. Place them in `/public/images/{client-name}/`
3. Update config URLs to relative paths: `/images/{client-name}/image-name.jpg`

For now: **Leave external URLs as-is, mention to user that images can be moved to `/public/images/` later.**

---

### Step 6: Initialize Git Repo

```bash
cd ~/Documents/Github/{repo-name}
rm -rf .git
git init
git branch -m main
git add .
git commit -m "init: {repo-name}"
```

Output:
```
[main (root-commit) {hash}] init: {repo-name}
 XX files changed, XXXX insertions(+)
```

---

### Step 7: Validate TypeScript

Run type check to ensure no errors:
```bash
cd ~/Documents/Github/{repo-name}
npx tsc --noEmit
```

**Expected output:** No errors

---

### Step 8: Output Summary

Print a clear summary to the user:

```
✅ Website generated successfully!

Repository: {repo-name}
Location: ~/Documents/Github/{repo-name}
Business: {client-name} ({business-type})
Primary Color: {primary-color}
Design Variant: {design-variant}

Next steps:
1. Review config/business.ts for any corrections
2. Replace postimg URLs with local images (optional but recommended):
   - Add images to /public/images/{client-name}/
   - Update config/business.ts URLs
3. Create GitHub repo: github.com/new → {repo-name}
4. Push to GitHub:
   git remote add origin git@github.com:{username}/{repo-name}.git
   git push -u origin main
5. Import on Vercel: vercel.com/new → {repo-name}
6. Get .vercel.app URL and test

Generated at: {timestamp}
```

## Common Issues & Fixes

| Issue | Solution |
|-------|----------|
| Business type is invalid | Check BRIEF.md — use only: pension, hotel, cottage, restaurant, salon, generic |
| Phone number has spaces/+ | Remove them — example: "0269 227 800" → "0269227800" |
| Primary color is not hex | Use imagecolorpicker.com on their logo |
| Postimg URLs are slow | User will move to /public/images later — this is OK for demo |
| WhatsApp number is incomplete | Ensure full Romanian format: 40722123456 (no spaces, no +) |
| Maps embed has full iframe tag | Extract just the URL from src="..." |
| Rooms/testimonials have too many | Keep all of them — no limit in template |
| TypeScript errors on build | Check config/business.ts syntax — all required fields present? |

---

## After Generation

- User manually creates repo on GitHub
- User pushes from local
- User imports in Vercel
- Site deploys automatically
- User gets .vercel.app URL
- Site is live

**Note:** Everythign before GitHub/Vercel is automated by these instructions. Once it's pushed, Vercel handles deployment.
