# Website Generator Agent (Old)

Automate the creation of a new client website repo from BRIEF.md data.

## Workflow

### Input

- `BRIEF.md` file with client data fully or partially filled in

### Output

- New GitHub-ready repo directory: `~/Documents/Github/website-{type}-{client-slug}`
- Initialized Git repo with one commit: `init: website-{type}-{client-slug}`
- Populated `config/business.ts` with all client data
- Summary with next steps

### Process

1. **Validate BRIEF.md**
   - Check all required fields are present
   - Validate business type (pension | hotel | cottage | restaurant | salon | generic)
   - Validate color format (hex)
   - Show user any missing required fields

2. **Generate Metadata**
   - Create repo name: `website-{businessType}-{slugifiedName}`
   - Assign design variant (classic by default, or per-type if specified)

3. **Copy Template**
   - Copy entire `/Users/sufanaadrian/Documents/Github/website-template` to new location
   - Remove old `.git` history
   - Create `/public/images/{client-slug}` directory

4. **Generate config/business.ts**
   - Parse BRIEF.md line by line
   - Map sections to TypeScript config object
   - Handle optional fields (remove if empty)
   - Ensure all strings are properly quoted and escaped
   - Validate TypeScript syntax

5. **Commit**
   - `git init` new repo
   - `git branch -m main`
   - `git add .`
   - `git commit -m "init: website-{type}-{client-slug}"`

6. **Validate**
   - Run `npx tsc --noEmit` to check TypeScript
   - Report any errors

7. **Summary**
   - Show repo path, business name, color, variant
   - List next steps (GitHub push, Vercel import)

## BRIEF.md Field Mapping

### Required

- Name → `name`
- Business Type → `businessType`
- Phone → `phone`
- Email → `email`
- Address → `address`
- City → `city`
- Primary Color → `primaryColor`

### Optional (omit if empty)

- Tagline → `tagline`
- Description → `description`
- Hours → `hours`
- Hero image → `heroImageUrl`, `heroImageAlt`
- Booking URLs → `bookingComUrl`, `airbnbUrl`
- Social links → `socialLinks`
- WhatsApp → `whatsapp`
- Maps embed → `googleMapsEmbedUrl`
- Services → `services[]`
- Rooms → `rooms[]`
- Testimonials → `testimonials[]`
- FAQ → `faq[]`
- Gallery → `gallery[]`
- Bullet points → `bulletPoints[]`
- Stats → `stats[]`

## Repository Naming Convention

Format: `website-{businessType}-{businessNameSlug}`

Rules:

- Lowercase
- Replace spaces with hyphens
- Remove special characters (accents, apostrophes, etc.)
- Max 50 characters
- No underscores

Examples:

- "Pension GreenHouse" → `website-pension-greenhouse`
- "Azur Restaurant" → `website-restaurant-azur`
- "The Fade Room" → `website-salon-the-fade-room`
- "Cabana Vânătorilor" → `website-pension-cabana-vanatorilor`

---

## Design Variant Logic

Default: `"classic"`

Per business type (override if needed):

- `pension` → `classic` (warm, traditional)
- `hotel` → `dark` (upscale aesthetic)
- `restaurant` → `dark` (elegant)
- `cottage` → `classic` (cozy)
- `salon` → `bold` (modern)
- `generic` → `classic` (neutral)

Can be specified in BRIEF.md as optional field: `**Design Variant:** dark`

## Image Handling

### Current (external URLs)

Images in BRIEF.md can be:

- Unsplash URLs (free, fast)
- Postimg.cc URLs (free upload, slower)
- Any direct image URL

Keep as-is in config during generation.

### Later (local images)

User can move images to `/public/images/{client-slug}/` and update config:

```ts
heroImageUrl: "/images/greenhouse/hero.jpg";
gallery: [
  "/images/greenhouse/gallery-1.jpg",
  "/images/greenhouse/gallery-2.jpg",
];
```

During generation: **Don't enforce this — optional improvement.**

## Error Handling

| Error                    | Action                                                      |
| ------------------------ | ----------------------------------------------------------- |
| Required field missing   | Show which fields, ask user to fill BRIEF.md, don't proceed |
| Invalid business type    | Show valid options, ask user to fix BRIEF.md                |
| Invalid hex color        | Use tool: imagecolorpicker.com, ask user to fix             |
| WhatsApp format wrong    | Show format (40722000000), ask user to fix                  |
| Maps iframe has full tag | Extract just the URL from src=""                            |
| TypeScript errors        | Show error, check config syntax, offer to fix               |
| Postimg URL is broken    | Note it, suggest local images later, don't block            |

## Next Steps for User (After Generation)

1. Review `config/business.ts` for correctness
2. Create GitHub repo: `github.com/new` → name it same as directory
3. Push:
   ```bash
   cd ~/Documents/Github/website-{type}-{client-slug}
   git remote add origin git@github.com:{username}/website-{type}-{client-slug}.git
   git push -u origin main
   ```
4. Import in Vercel: `vercel.com/new` → select repo → deploy
5. Get `.vercel.app` URL
6. (Later: replace postimg URLs with local images in `/public/images/`)

## Troubleshooting

**"BRIEF.md is empty"**
→ Ask user to fill it in first

**"Config doesn't look right"**
→ Compare with `.instructions.md` mapping table

**"Images not showing on site"**
→ Check image URLs in config/business.ts — must be valid URLs or `/images/...` paths

**"Can't push to GitHub"**
→ Ensure remote is set: `git remote -v` should show origin

**"Vercel import failed"**
→ Ensure repo is on GitHub and GitHub account is connected to Vercel
